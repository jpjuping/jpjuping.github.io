计算机网络基础

OSI（开放性的通信系统互连参考模型）网络七层协议和五层协议

OSI网络五层协议

1. 应用层 

1.1 与其他计算机进行通讯的一个应用

1.2 HTTP、FTP、NFS、SMTP、TELNET等

1.3 域名系统DNS
    域名解析过程：
        + 主机向本地域名服务器的查询一般都采用递归查询
        + 本地域名服务器向根域名服务器的查询采用迭代查询

2. 传输层 

2.1 TCP、UDP
    TCP: 提供面向连接的服务
        + TCP是面向连接的
        + 每一条TCP连接只能有两个端点，只能点对点
        + TCP提供可靠的交付，提供传送的数据无差错，不丢失，不重复，按序到达

    UDP: 传送数据之前不需要先建立连接
        + UDP是无连接的，发送数据之前不需要建立连接，减少了开销和发送数据之前的时延
        + UDP是面向报文的
        + UDP没有拥塞控制

3. 网络层 

3.1 IP协议
    与IP协议配套是用的三个协议：
        + 地址解析协议ARP（Address Resolution Protocol）
        + 网际控制报文协议ICMP（Internet Control Protocol）
        + 网际组管理协议IGMP（Internet Group Management Protocol）

3.2 网络层为建立网络连接和为上层提供服务

3.3 网络连接设备： 
    + 网络层以上使用的中间设备： 网关（gateway）
    + 网络层使用的中间设备：路由器（router）
    + 数据链路层使用的中间设备： 网桥或桥接器（bridge）
    + 物理层使用的中间设备： 转发器（repeater）

3.4 IP地址：
    IP地址的编址方法三个阶段：
        + 网络号(标识主机或路由器所连接到的网络)+主机号（标识主机或路由器）
        + 子网的划分
        + 构成超网

4. 数据链路层 

4.1 定义了单个链路上如何传输数据。这些协议与被讨论的各种介质有关。

4.2 数据链路层产品：最常见的当属网卡和网桥

4.3 链路层的数据传输单元是“帧”，协议不同帧的长短和界面也有差别

4.4 主要功能
    4.4.1 封装成帧： 在一段数据的前后分别添加首部和尾部，这样就构成了帧，首部和尾部的作用就是帧定界
    4.4.2 透明传输
    4.4.3 差错检测：bit传输过程中可能会出现差错，1变成0，目前在数据链路层广泛使用了循环冗余检测CRC( Cyclic Redundancy Check)的检错技术

5. 物理层 

5.1 物理层产品：计算机，终端，光纤等

5.2 主要功能：
    5.2.1 为传输数据提供数据通路

    5.2.2 传输数据，形成是和数据传输的实体，满足点对点，一点到多点，并行，串行，同步，异步等

网络安全问题概述

计算机网络面临的安全威胁

1. 被动攻击
    攻击者从网络上窃听他人的通讯内容，称为流量分析

2. 主动攻击
    2.1 篡改 攻击者故意篡改网络上传送的报文
    2.2 恶意程序 计算机病毒 木马等
    2.3 拒绝服务 攻击者向因特网上某个服务器不停的发送大量分组，是服务器停止提供服务。若成千上百个网站集中攻击一个网站，称为分布式拒绝服务DDOS

Markdown文档格式的学习

1. 什么是markdown?

简单来说，markdown是一个翻译器，在写文档的时候通过简单的几个标记，代表不同的样式格式。markdown能够把这些标记翻译为浏览器能够识别的排版规则。

2. 有必要使用markdown吗？

markdown最核心的优势在于：在不打算写作思路的前提下进行排版，同时解放你的鼠标。从而极大的提高效率

3. Markdown能用来做什么？

说明文档，文案，博客，消息通知等

4. Markdown怎么使用？在文档中的用法？

安装：
